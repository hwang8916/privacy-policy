2. 수집하는 개인정보 항목

저희 앱은 다음과 같은 개인정보를 수집합니다:

2.1 사용자가 직접 제공하는 정보

  • 계정 정보: 이메일 주소 (이메일 회원가입), Google 계정 정보 (Google 로그인)
  • 사용자 생성 데이터:
    • 습관 정보: 습관 제목, 아이콘, 색상, 설명
    • 습관 완료 기록: 완료 날짜, 시간, 연속 달성 여부
    • 타이머 설정: 작업 시간, 휴식 시간, 세션 수
    • 타마고치/캐릭터 데이터: 캐릭터 이름, 진화 단계, 보유 스티커
  • 앱 설정: 테마, 언어, 알림 선호도 등 사용자 지정 설정

2.2 자동으로 수집되는 정보

  • 기기 정보:
    • Firebase Cloud Messaging (FCM) 토큰
    • 기기 OS 및 버전 (Android)
    • 앱 버전
  • 사용 분석 데이터:
    • 사용자 ID (Firebase UID)
    • 로그인 방법 (이메일, Google)
    • 앱 사용 이벤트 (화면 방문, 기능 사용)
    • 통계 데이터 (습관 수, 컬렉션 수, 완료율)
  • 오류 및 성능 데이터:
    • 앱 충돌 정보 (Firebase Crashlytics)
    • 오류 로그 및 스택 트레이스
    • 성능 메트릭

2.3 권한을 통해 수집되는 정보

앱이 정상 작동하기 위해 다음 Android 권한을 요청합니다:

  • POST_NOTIFICATIONS: 일일 습관 리마인더 및 타이머 알림 전송
  • SCHEDULE_EXACT_ALARM: 정확한 시간에 알림 발송
  • RECEIVE_BOOT_COMPLETED: 기기 재부팅 후 알림 복원
  • VIBRATE: 알림 시 진동 피드백

3. 정보 수집 방법

  • 직접 입력: 회원 가입, 로그인, 습관 생성 및 편집 등
  • 자동 수집: Firebase SDK를 통한 자동 데이터 수집
  • 권한 허용: 사용자가 앱에 부여한 권한(알림, 알람, 진동)을 통한 수집

4. 정보 사용 목적

저희는 수집한 개인정보를 다음 목적으로만 사용합니다:

  • 서비스 제공: 앱의 핵심 기능 제공 (습관 추적, 타마고치 게임, 타이머)

  • 데이터 동기화: 여러 기기 간 데이터 동기화 (Cloud Firestore 사용)

  • 앱 개선 및 분석: 사용 패턴 분석, 기능 개선 (Firebase Analytics)
  • 오류 진단 및 수정: 앱 충돌 및 버그 분석 (Firebase Crashlytics)
  • 법적 요건 충족: 법률 준수 및 분쟁 해결

5. 제3자 서비스 (Firebase)

저희 앱은 사용자 데이터를 더 나은 서비스 제공을 위해 다음 Google Firebase 서비스에 공유합니다:

🔐 Firebase Authentication (인증)

목적: 안전한 로그인 및 계정 관리

수집 데이터: 이메일 주소, Google 계정 정보, 사용자 UID

보관 기간: 계정 삭제 시까지

정책 링크: Firebase Privacy Policy | Google Privacy Policy

💾 Cloud Firestore (데이터베이스)

목적: 사용자 습관, 타마고치, 설정 데이터 저장 및 동기화

수집 데이터:

  • 습관 정보 및 완료 기록
  • 타마고치 캐릭터 데이터
  • 타이머 설정
  • 앱 설정

보관 기간: 계정 삭제 시까지

정책 링크: Firebase Privacy Policy

📢 Firebase Cloud Messaging (FCM)

목적: 푸시 알림 전송 (일일 습관 리마인더, 타이머 알림)

수집 데이터: FCM 토큰, 기기 정보

보관 기간: 앱 설치 시까지

정책 링크: Firebase Privacy Policy

📊 Firebase Analytics (분석)

목적: 앱 사용 패턴 분석 및 기능 개선

수집 데이터:

  • 사용자 ID
  • 로그인 방법
  • 화면 방문 이벤트
  • 기능 사용 통계
  • 습관 수, 컬렉션 수 등의 통계

보관 기간: 기본적으로 14개월 (Google의 정책에 따름)

정책 링크: Firebase Privacy Policy | Google Analytics for Firebase

🐛 Firebase Crashlytics (오류 분석)

목적: 앱 충돌 및 오류 분석으로 안정성 개선

수집 데이터:

  • 앱 충돌 정보
  • 오류 로그 및 스택 트레이스
  • 기기 정보
  • 성능 메트릭

보관 기간: 기본적으로 90일 (Google의 정책에 따름)

정책 링크: Firebase Privacy Policy

🔗 Google Sign-In

목적: Google 계정을 통한 편리한 로그인

수집 데이터: Google 계정 정보, 프로필 정보

보관 기간: 계정 삭제 시까지

정책 링크: Google Privacy Policy

⚠️ 중요: Google Firebase는 미국 소재 회사입니다. 따라서 사용자 데이터는 Google의 데이터 처리 약관에 따라 관리되며, 미국 법에 따를 수 있습니다. Google의 국제 데이터 이전 정책은 Google Privacy Policy에서 확인할 수 있습니다.

7. 데이터 보안

저희는 사용자의 개인정보 보호를 최우선으로 생각하며, 다음과 같은 보안 조치를 취하고 있습니다:

  • HTTPS 암호화: 모든 데이터 전송은 SSL/TLS 암호화 프로토콜을 사용합니다.
  • Firebase Security Rules: Cloud Firestore는 Firebase Security Rules로 보호되며, 사용자는 자신의 데이터에만 접근할 수 있습니다.
  • 인증 기반 접근: 모든 사용자는 로그인을 통해 자신의 데이터에 접근할 수 있습니다.
  • Google의 보안: Google Firebase는 Google의 엔터프라이즈급 보안 시설을 사용합니다.
  • 데이터 암호화: Google Firestore는 저장 시 데이터를 암호화합니다.

주의: 어떤 온라인 서비스도 100% 완벽한 보안을 보장할 수 없습니다. 사용자는 자신의 계정 정보(비밀번호, 로그인 정보)를 안전하게 보관해야 합니다.
