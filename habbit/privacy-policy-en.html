1. Preamble

Hello, this is the Tok Tok: Habit & Hatch team. This Privacy Policy describes how we collect, use, and manage personal information through our mobile application 'Tok Tok: Habit & Hatch' (hereafter "the App").

This app is a gamified habit tracker where users track daily habits and grow a digital tamagotchi character upon completion. It also provides a Pomodoro timer and focus features.

App Information:

  • App Name: Tok Tok: Habit & Hatch
  • Package Name: com.toktok.habbit
  • Version: 1.0.0+1
  • Developer: Tok Tok: Habit & Hatch team
  • Contact: hwang8916@gmail.com

2. Personal Information Items Collected

Our app collects the following personal information:

2.1 Information Directly Provided by Users

  • Account Information: Email address (email signup), Google account information (Google login)
  • User-Generated Data:
    • Habit Information: Habit title, icon, color, description
    • Habit Completion Records: Completion date, time, streak status
    • Timer Settings: Work time, break time, number of sessions
    • Tamagotchi/Character Data: Character name, evolution stage, collected stickers
  • App Settings: Theme, language, notification preferences, and other user-customized settings

2.2 Automatically Collected Information

  • Device Information:
    • Firebase Cloud Messaging (FCM) token
    • Device OS and version (Android)
    • App version
  • Usage Analytics Data:
    • User ID (Firebase UID)
    • Login method (email, Google)
    • App usage events (screen visits, feature usage)
    • Statistical data (number of habits, collections, completion rate)
  • Error and Performance Data:
    • App crash information (Firebase Crashlytics)
    • Error logs and stack traces
    • Performance metrics

2.3 Information Collected Through Permissions

The app requests the following Android permissions to function properly:

  • POST_NOTIFICATIONS: Send daily habit reminders and timer notifications
  • SCHEDULE_EXACT_ALARM: Send notifications at precise times
  • RECEIVE_BOOT_COMPLETED: Restore notifications after device reboot
  • VIBRATE: Vibration feedback for notifications

3. Information Collection Methods

  • Direct Input: User registration, login, habit creation and editing, etc.
  • Automatic Collection: Automatic data collection via Firebase SDK
  • Permission Grant: Collection through permissions granted to the app (notifications, alarms, vibration)
  • Third-Party Services: Google Firebase and Google Sign-In services

4. Purpose of Information Use

We use collected personal information only for the following purposes:

  • Service Provision: Provide core app features (habit tracking, tamagotchi game, timer)
  • Account Management: Login, user registration, account security
  • Data Synchronization: Data synchronization across multiple devices (using Cloud Firestore)
  • Push Notifications: Daily habit reminders and timer notifications (Firebase Cloud Messaging)
  • App Improvement and Analytics: Usage pattern analysis and feature improvement (Firebase Analytics)
  • Error Diagnosis and Fixing: App crash and bug analysis (Firebase Crashlytics)
  • Legal Compliance: Legal compliance and dispute resolution

6. Data Retention and Deletion

6.1 Data Retention Period

  • Active Account: Retained while user is using the app
  • After Account Deletion:
    • Firebase Authentication: Deleted immediately
    • Cloud Firestore: Deleted within 30 days of account deletion
    • Firebase Analytics: Automatically deleted after 14 months
    • Firebase Crashlytics: Automatically deleted after 90 days

6.2 Data Deletion Method

Users can delete their account and data in the following ways:

Account and Data Deletion Request:

Please contact us via email with the following information:

  • Your email address
  • Your username or UID registered in the app
  • Subject: "Account and Data Deletion Request"

Email: hwang8916@gmail.com

Response Time: Within 7 days of request receipt

6.3 In-App Data Deletion

Users can delete the following data anytime within the app:

  • Habit data
  • Tamagotchi/character data
  • App settings

7. Data Security

We prioritize protecting your personal information and implement the following security measures:

  • HTTPS Encryption: All data transmission uses SSL/TLS encryption protocol.
  • Firebase Security Rules: Cloud Firestore is protected by Firebase Security Rules, allowing users to access only their own data.
  • Authentication-Based Access: All users access their data through login.
  • Google's Security: Google Firebase uses Google's enterprise-grade security facilities.
  • Data Encryption: Google Firestore encrypts data at rest.

Note: No online service can guarantee 100% perfect security. Users should keep their account information (passwords, login credentials) safe.
