1. 前文

こんにちは。Tok Tok: Habit & Hatch チームです。本プライバシーポリシーは、当チームが提供するモバイルアプリケーション「Tok Tok: Habit & Hatch」(以下「本アプリ」)を通じて収集、使用、および管理される個人情報に関する内容を説明しています。

本アプリは、ユーザーが日々の習慣を追跡し、完了することでデジタルたまごっちキャラクターを成長させるゲーミフィケーション習慣トラッカーです。さらに、ポモドーロタイマーと集中機能を提供しています。

アプリ情報:

  • アプリ名: Tok Tok: Habit & Hatch
  • パッケージ名: com.toktok.habbit
  • バージョン: 1.0.0+1
  • 開発者: Tok Tok: Habit & Hatch チーム
  • お問い合わせ: hwang8916@gmail.com

2. 収集される個人情報

本アプリは以下の個人情報を収集します:

2.1 ユーザーが直接提供する情報

  • アカウント情報: メールアドレス (メール登録)、Google アカウント情報 (Google ログイン)
  • ユーザー生成データ:
    • 習慣情報: 習慣タイトル、アイコン、色、説明
    • 習慣完了記録: 完了日時、時刻、連続達成ステータス
    • タイマー設定: 作業時間、休憩時間、セッション数
    • たまごっち/キャラクターデータ: キャラクター名、進化段階、収集したシール
  • アプリ設定: テーマ、言語、通知設定、その他のユーザーカスタマイズ設定

2.2 自動的に収集される情報

  • デバイス情報:
    • Firebase Cloud Messaging (FCM) トークン
    • デバイス OS およびバージョン (Android)
    • アプリバージョン
  • 使用分析データ:
    • ユーザー ID (Firebase UID)
    • ログイン方法 (メール、Google)
    • アプリ使用イベント (画面訪問、機能使用)
    • 統計データ (習慣数、コレクション数、完了率)
  • エラーおよびパフォーマンスデータ:
    • アプリクラッシュ情報 (Firebase Crashlytics)
    • エラーログおよびスタックトレース
    • パフォーマンスメトリクス

2.3 許可を通じて収集される情報

本アプリが正常に機能するために以下の Android 権限をリクエストします:

  • POST_NOTIFICATIONS: 日々の習慣リマインダーとタイマー通知の送信
  • SCHEDULE_EXACT_ALARM: 正確な時刻に通知を送信
  • RECEIVE_BOOT_COMPLETED: デバイス再起動後の通知復元
  • VIBRATE: 通知時の振動フィードバック

3. 情報収集方法

  • 直接入力: ユーザー登録、ログイン、習慣作成・編集など
  • 自動収集: Firebase SDK を通じた自動データ収集
  • 許可付与: アプリに付与された許可を通じた収集 (通知、アラーム、振動)
  • 第三者サービス: Google Firebase および Google Sign-In サービス

4. 情報の利用目的

当チームは収集した個人情報を以下の目的のみで使用します:

  • サービス提供: アプリの中核機能提供 (習慣追跡、たまごっちゲーム、タイマー)
  • アカウント管理: ログイン、ユーザー登録、アカウントセキュリティ
  • データ同期: 複数デバイス間でのデータ同期 (Cloud Firestore 使用)
  • プッシュ通知: 日々の習慣リマインダーとタイマー通知 (Firebase Cloud Messaging)
  • アプリ改善と分析: 使用パターン分析、機能改善 (Firebase Analytics)
  • エラー診断と修正: アプリクラッシュとバグ分析 (Firebase Crashlytics)
  • 法的義務の遵守: 法律遵守と紛争解決

5. 第三者サービス (Firebase)

本アプリはより良いサービス提供のため、以下の Google Firebase サービスとユーザーデータを共有します:

🔐 Firebase Authentication

目的: 安全なログインとアカウント管理

収集データ: メールアドレス、Google アカウント情報、ユーザー UID

保管期間: アカウント削除時まで

ポリシーリンク: Firebase プライバシーポリシー | Google プライバシーポリシー

💾 Cloud Firestore (データベース)

目的: ユーザー習慣、たまごっち、設定データの保存と同期

収集データ:

  • 習慣情報と完了記録
  • たまごっちキャラクターデータ
  • タイマー設定
  • アプリ設定

保管期間: アカウント削除時まで

ポリシーリンク: Firebase プライバシーポリシー

📢 Firebase Cloud Messaging (FCM)

目的: プッシュ通知送信 (日々の習慣リマインダー、タイマー通知)

収集データ: FCM トークン、デバイス情報

保管期間: アプリアンインストール時まで

ポリシーリンク: Firebase プライバシーポリシー

📊 Firebase Analytics

目的: アプリ使用パターンの分析と機能改善

収集データ:

  • ユーザー ID
  • ログイン方法
  • 画面訪問イベント
  • 機能使用統計
  • 習慣数、コレクション数などの統計

保管期間: 通常 14 ヶ月 (Google のポリシーに従う)

ポリシーリンク: Firebase プライバシーポリシー | Firebase Analytics

🐛 Firebase Crashlytics (エラー分析)

目的: アプリクラッシュとエラー分析による安定性改善

収集データ:

  • アプリクラッシュ情報
  • エラーログおよびスタックトレース
  • デバイス情報
  • パフォーマンスメトリクス

保管期間: 通常 90 日 (Google のポリシーに従う)

ポリシーリンク: Firebase プライバシーポリシー

🔗 Google Sign-In

目的: Google アカウントを通じた便利なログイン

収集データ: Google アカウント情報、プロフィール情報

保管期間: アカウント削除時まで

ポリシーリンク: Google プライバシーポリシー

⚠️ 重要: Google Firebase は米国を拠点とする企業です。したがって、ユーザーデータは Google のデータ処理約款に従って管理され、米国法の対象となる可能性があります。Google の国際データ転送ポリシーは Google プライバシーポリシーで確認できます。

6. データ保管と削除

6.1 データ保管期間

  • アクティブなアカウント: ユーザーがアプリを使用している間、保管され続ける
  • アカウント削除後:
    • Firebase Authentication: 即座に削除
    • Cloud Firestore: アカウント削除後 30 日以内に削除
    • Firebase Analytics: 14 ヶ月後に自動削除
    • Firebase Crashlytics: 90 日後に自動削除

6.2 データ削除方法

ユーザーは以下の方法でアカウントとデータを削除できます:

7. データセキュリティ

当チームはユーザーの個人情報保護を最優先と考え、以下のセキュリティ対策を実施しています:

  • HTTPS 暗号化: すべてのデータ転送は SSL/TLS 暗号化プロトコルを使用します。
  • Firebase セキュリティルール: Cloud Firestore は Firebase セキュリティルールで保護され、ユーザーは自身のデータのみにアクセスできます。
  • 認証ベースのアクセス: すべてのユーザーはログインを通じてデータにアクセスします。
  • Google のセキュリティ: Google Firebase は Google のエンタープライズグレードセキュリティ施設を使用します。
  • データ暗号化: Google Firestore は保管時にデータを暗号化します。

注意: オンラインサービスは 100% の完璧なセキュリティを保証できません。ユーザーはアカウント情報 (パスワード、ログイン認証) を安全に管理してください。
